Project Overview

I migrated the some Payrexx frontend codebase from Vue 2 (Options API) to Vue 3 (Composition API) to improve performance, code reusability, and long-term maintainability. The migration followed a structured workflow covering analysis, refactoring, dependency updates, bug fixing, testing, and final deployment.

This comprehensive migration ensured the frontend remained modern, scalable, and aligned with current Vue.js best practices while maintaining full backward compatibility with existing backend systems.

Key Contributions

  • Converted large and complex components from Options API to Composition API, ensuring cleaner logic separation and improved reusability
  • Replaced deprecated Vue 2 lifecycle hooks and patterns with modern Vue 3 equivalents
  • Removed unused code, optimized component structure, and reduced bundle size
  • Updated core dependencies (Vue Router, Vuex/Pinia, Axios, Webpack loaders, plugins)
  • Implemented linting and formatting consistency across modules
  • Coordinated with the backend team (PHP/Laravel) for API compatibility

Major Challenges & Solutions

1. Breaking Changes in Vue 3 Lifecycle Hooks

Issue: beforeDestroy and destroyed hooks were removed; some watchers stopped working.
Fix:
  • Rewrote old lifecycle logic using onMounted, onUnmounted, and watchEffect
  • Refactored watchers to watch() with cleanup functions

2. Problems with Third-Party Vue 2 Plugins

Issue: Several Vue 2 plugins were incompatible with Vue 3.
Fix:
  • Replaced outdated plugins with Vue 3 alternatives
  • Where no replacements existed, rewrote small plugin logic manually using composables

3. Vuex Store Compatibility Issues

Issue: Old Vuex modules were tightly coupled and broke when migrating.
Fix:
  • Modularized store into smaller pieces
  • Started transitioning shared logic to Pinia for better type safety and structure

4. Webpack & Build Errors After Dependency Updates

Issue: Build failed due to outdated loaders and deprecated configs.
Fix:
  • Updated Webpack loaders (babel-loader, vue-loader, css-loader)
  • Cleaned old polyfills and optimized assets
  • Fixed compatibility issues with ESLint, Babel presets, and PostCSS

5. UI Regression & Styles Breaking

Issue: Some components rendered differently after migration.
Fix:
  • Performed a full UI regression test across modules
  • Fixed scoping issues using :deep() and proper scoped CSS usage
  • Standardized CSS variables and component-level styling

6. Communication Challenges Across Teams

Issue: Migration needed coordination with multiple internal teams.
Fix:
  • Documented all changes clearly
  • Provided weekly updates and created checklists for backend/front-end compatibility
